# feat: Fumadocs Documentation Site for Skillbox

## Overview

Build a developer-friendly documentation site using [Fumadocs](https://fumadocs.dev) (v16.x, Next.js App Router) in `docs-site/` at the repository root. The site serves both AI/agent developers and platform operators with a modern dark theme, progressive content structure, and auto-generated API reference from OpenAPI specs.

## Problem Statement

Skillbox has rich documentation scattered across `README.md`, `docs/ARCHITECTURE.md`, `docs/API.md`, `docs/SKILL-SPEC.md`, and `docs/SKILL-CREATOR-GUIDE.md` â€” but no unified, browsable, searchable documentation site. The existing `docs/API.md` is outdated (missing 16 of 27 endpoints). Developers integrating Skillbox into agent workflows have no guided learning path.

## Proposed Solution

A standalone Fumadocs site in `docs-site/` with:
- **5 content sections**: Getting Started, Concepts, Guides, API Reference, SDKs
- **Auto-generated API reference** via swaggo annotations â†’ Swagger 2.0 â†’ OpenAPI 3.x conversion â†’ fumadocs-openapi rendering
- **Dark theme** matching the landing page design tokens (OKLCH green accent, Inter + JetBrains Mono fonts)
- **Built-in search** via Orama (no external dependencies)

## Known Limitations and Workarounds

### swaggo generates Swagger 2.0, not OpenAPI 3.x

**Workaround:** Pipeline includes `swagger2openapi` conversion step. This is a build-time transformation, not runtime. The `make swagger` target handles both steps.

### `ListSkills` hardcodes `mode: "executable"`

**Workaround:** Document in the API reference that the `mode` field in list responses defaults to `"executable"`. Recommend fetching individual skills via `GET /v1/skills/:name/:version` for accurate mode information.

### Conditional endpoint registration

**Workaround:** Document all endpoints in the OpenAPI spec. Add a note in the API Reference introduction page explaining that Files, Sessions, and Sandbox endpoints require their respective backing services (MinIO, OpenSandbox) to be configured. Endpoints return 404 when their service is unavailable.

### SDK distribution

The Go SDK is importable as a Go module (`go get github.com/devs-group/skillbox/sdks/go`). The Python SDK is currently a single vendorable file (not on PyPI). Document both patterns explicitly.

### No API key management endpoint

API keys are provisioned via `scripts/seed-apikey.sh` (direct database insertion). Document this in Quick Start and Authentication concept page. Note that a management API is planned.
